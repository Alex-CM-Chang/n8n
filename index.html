二、模型路由決策表（n8n 路由節點邏輯）

n8n 在 AI 處理層前插入一個 Router 節點，根據任務類型、粉絲等級、敏感度三個維度，決定呼叫哪個引擎。以下是完整路由規則：

任務類型 觸發條件 指定模型 理由
粉絲聊天 — 一般粉 (<$20) fan_tier = basic Qwen3 30B 批量高頻、審查限制少、中英文OK
粉絲聊天 — 活躍粉 ($20-100) fan_tier = active Qwen3 72B 升級推理能力，仍零邊際成本
粉絲聊天 — VIP ($100-500) fan_tier = vip Claude Haiku 語感品質要求高，值得付費
粉絲聊天 — 鯨魚 (>$500) fan_tier = whale Claude Haiku 高客單價，品質優先
跨平台行銷文案 workflow = content_gen Qwen3 30B 成人相關文案審查限制更少
ComfyUI Prompt 生成 workflow = image_gen Gemma 4 26B 結構化輸出強，prompt engineering OK
內容審核 / 違規詞過濾 node = safety_filter Gemma 4 E4B 官方支援此場景，輕量快速
中英翻譯 task = translate Gemma 4 26B 140+ 語言原生，256K context
粉絲情緒分析 / 分類 task = sentiment Gemma 4 26B 結構化推理任務，成本趨零
週報趨勢分析 + 建議 workflow = weekly_report Gemma 4 26B 長文分析 256K context，免費
月內容日曆規劃 workflow = content_calendar Gemma 4 26B 結構化 JSON 輸出，不需付費 API
高價值合約 / 法律文件摘要 doc_type = contract Claude Haiku 精確度要求高，fallback 保底
💡 n8n 路由實作方式：在 AI 節點前加一個 Switch 節點，讀取 fan_tier、task_type、workflow_id 三個變數，分叉到不同的 HTTP Request 節點（分別指向 Ollama 本機 API 或 Anthropic API）。Ollama 提供 OpenAI 相容 endpoint，換 base URL 即可，prompt 不用重寫。

MoE 架構只激活 4B 參數，實際 VRAM 需求低於 Dense 模型。RTX 4070 12GB 可跑 Q4 量化版（略需 offload）

七、關鍵技術備註（v2.0 新增事項）

Ollama 本機 API 設定

Ollama 提供 OpenAI 相容的 /v1/chat/completions endpoint，n8n 的 HTTP Request 節點只需修改 base URL，所有 prompt 結構完全相容，不需重寫工作流邏輯。

# 啟動 Ollama 並允許外部連接（n8n VPS → 本機工作站） OLLAMA_HOST=0.0.0.0 ollama serve # 拉取模型 ollama pull gemma4:27b # 16.5GB 下載 ollama pull qwen3:30b # 18GB 下載 # 確認 API 正常 curl http://localhost:11434/v1/chat/completions \ -H "Content-Type: application/json" \ -d '{"model":"gemma4:27b","messages":[{"role":"user","content":"test"}]}'

Gemma 4 成人內容限制處理

Gemma 4 預設有安全濾鏡，成人相關文字生成任務不建議直接使用官方版。建議：

  • 成人文案生成 → 改用 Qwen3（審查限制顯著更少）
  • Gemma 4 只負責邏輯分析、翻譯、審核等中性任務
  • 如需 Gemma 4 處理敏感文案，需使用 Hugging Face 上的社群去審查微調版（abliterated variants）

Claude Haiku vs Qwen3：中文語感比較

Claude Haiku 在繁體中文語感上仍有優勢，尤其是需要細膩情感表達的 VIP 聊天。Qwen3 的繁體中文品質近年大幅提升，一般粉絲的互動已難以區分差異。建議：初期全部先用 Qwen3 測試，只在粉絲投訴「回覆感覺不自然」時，才把該等級升回 Claude。

AI 合規揭露（不變，必須保留）

所有工作流無論使用哪個模型，都必須保留 AI 標記邏輯：聊天開頭自動插入 AI 輔助聲明、圖像加 #AIGenerated metadata、發文附 AI 標籤。此合規底線不因模型替換而改變。
